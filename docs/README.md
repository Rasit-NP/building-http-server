- [2026-07-21] string-view-request-line — `HttpRequest`의 request line 3필드(`method`/`path`/`version`)를 `std::string` 복사에서 `buffer_`를 가리키는 `std::string_view`(zero-copy)로 전환. `parseRequestLine()`의 `request_line = buffer_.substr(0, crlf)` 임시 복사를 제거하고 `buffer_` 위에서 직접 `find`, 위치를 `method_off_/len_`·`path_off_/len_`·`version_off_/len_` 멤버로 기록. dangling 회피 위해 `string_view` 대입은 `State::Done`으로 이동. 🔴 3건: (1) 대입 전 검증 — `isValid()`/`isValidVersion()`가 빈 view를 검사해 전 요청 `Error`(대입+검증을 `Done`으로 이동해 해소), (2) 여분 공백 검사 `buffer_.find(' ', sp2+1)`의 `crlf` 경계 누락으로 헤더 공백(`Host: a:8080`)을 request line 공백 3개로 오인(`sp3 != npos && sp3 < crlf`로 해소, 중간에 `||` 오타 경유), (3) 빈 토큰 판정이 `Done`으로 밀려 종료 CRLF 없는 불완전 요청이 `Error` 대신 `Incomplete`(length 기반 `!method_len_||!path_len_||!version_len_` 검사를 `parseRequestLine`에 두어 조기 판정). 실패 이력 8→3→3→1→0. macOS 로컬 파서 테스트 `tests` 21/21 통과·파서 TU `-Wall -Wextra -Wpedantic` 경고 0. `integration.*`는 `sys/epoll.h` 부재로 미실행 → CI 대기. 🟡 `string_view` 수명 계약(`buffer_` 재할당 시 dangling)·offset 멤버 지역화는 향후 과제 ([문서](2026-07-21-string-view-request-line.md))

- [2026-07-12] parser-hardening — 관대하게 통과시키던 `HttpRequestParser`에 4종 거부 규칙 도입.
